flowchart TD
SK000(Start) -- Obtain target country and item information --> SK001[Call 'query_shipping_status' API with the target country and item information, and get the returned status and shipping eligibility information]
SK001 -- Status code is 200 --> SK002[Check if the item can be shipped based on the returned isValid]
SK002 -- Item can be shipped --> SK003[Inform the user that the item can be shipped and state that the information is for reference only and the final decision is based on the destination customs clearance requirements]
SK002 -- Item cannot be shipped --> SK004[Call 'query_goods_list' API with the target country to get the list of supported item categories for that country]
SK004 --> SK005[Inform the user that the item cannot be shipped, provide the supported item categories for that country, and state that the information is for reference only and the final decision is based on the destination customs clearance requirements]
SK001 -- Status code is not 200 --> SK006[Inform the user that the query failed, state that the destination shipping restriction information was not found, and suggest that the user contact local DHL for further assistance]
SK000 -- Information provided by the user is incomplete --> SK007[Prompt the user to provide the necessary information]
