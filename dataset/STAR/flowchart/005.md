flowchart TD
    SK000(Start) -- Ask for user's name --> SK001[Ask the user for the doctor's name they wish to book an appointment with]
    SK001 -- Ask for the day --> SK002[Ask the user for the day they prefer for the appointment]
    SK002 -- Ask for the time --> SK003[Ask the user for the preferred start time of the appointment]
    SK003 -- Ask for symptoms --> SK004[Ask the user to describe their symptoms]
    SK004 -- Call doctor_schedule API --> SK005[Check the doctor's schedule for availability on the specified day and time]
    SK005 -- Doctor is available --> SK006[Inform the user about the booking availability]
    SK006 -- User confirms (yes) --> SK007[Book the appointment and inform the user that the booking was successful]
    SK006 -- User declines (no) --> SK001[Ask the user again for a different doctor's name and repeat the process]
    SK005 -- Doctor is not available --> SK008[Inform the user that the booking is unavailable]
    SK007 & SK008 -- Ask if user needs more assistance --> SK009[Ask the user if there is anything else they need assistance with]
    SK009 -- Yes, needs more assistance --> SK001
    SK009 -- No more assistance needed --> SK010[End the process and thank the user]
